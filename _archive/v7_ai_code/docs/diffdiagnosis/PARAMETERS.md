# Параметры проекта AIM/DiffDiagnosis

| Раздел | Параметр | Значение | Описание |
|--------|----------|----------|----------|
| Engine | `LLM_RERANK_TOP_K` | 10 | Количество кандидатов, возвращаемых LLM для переранжирования |
| Engine | `MIN_PROBABILITY_TO_REPORT` | 0.02 | Минимальная вероятность нозологии для включения в итоговый дифференциальный ряд |
| Engine | `MAX_DIFFERENTIAL_OUTPUT` | 20 | Максимальное количество диагнозов в финальном списке |
| Engine | `ENGINE_TIMEOUT_MS` | 5000 | Тайм-аут выполнения диагностического ядра (мс) |
| API | `API_PORT` | 8765 | Порт HTTP-сервера (Rust backend, axum) |
| API | `API_RATE_LIMIT_RPM` | 60 | Ограничение числа запросов в минуту |
| API | `API_MAX_REQUEST_KB` | 256 | Максимальный размер входящего запроса (КБ) |
| LLM | `LLM_PROVIDER` | deepseek | Поставщик LLM (через AIM/llm.py) |
| LLM | `LLM_TEMPERATURE` | 0 | Температура для генерации (детерминированный режим) |
| LLM | `LLM_MAX_TOKENS` | 4096 | Максимальное число токенов в ответе LLM |
| Phoenix | `PHOENIX_PORT` | 4000 | Порт Phoenix LiveView frontend |
| Phoenix | `LIVEVIEW_HEARTBEAT_MS` | 30000 | Интервал heartbeat LiveView (мс) |
| Метрики качества (target) | `top-1 accuracy` | ≥ 0.55 | Доля случаев, где первый диагноз совпадает с золотым стандартом |
| Метрики качества (target) | `top-3 accuracy` | ≥ 0.80 | Доля случаев, где истинный диагноз входит в топ-3 |
| Метрики качества (target) | `can't-miss-miss-rate` | ≤ 0.02 | Частота пропуска критического (жизнеугрожающего) диагноза |
| Гранты / дедлайны / бюджет | — | TBD (будет уточнено) | Финансовые и временны́е рамки подпроекта |
| Версии стека | Rust | ≥ 1.78 | Минимальная версия компилятора Rust |
| Версии стека | Elixir | ≥ 1.17 | Минимальная версия Elixir |
| Версии стека | Phoenix | ≥ 1.7 | Минимальная версия Phoenix Framework |
| Версии стека | Node.js (assets) | ≥ 20 | Минимальная версия Node.js для сборки фронтенда |
