## Архитектурные решения

### 2026-03-28: Перестройка с нуля на базе CONCEPT.md v6.0

**Решение:** Полная перестройка AIM с нуля на основе архитектурного видения v6.0.

**Причина:** Предыдущие версии накапливали технический долг. CONCEPT.md прошёл через peer review и содержит все критические улучшения.

**Ключевые выборы:**
- SQLite для локальной разработки → PostgreSQL для production
- DeepSeek API вместо Ollama для всех задач (качество + скорость)
- `llm.py` как единственная точка входа для всех LLM-вызовов
- `config.py` как единственный источник конфигурации
- `i18n.py` как источник всех строк (4 языка)

---

### 2026-03-28: LLM выбор — DeepSeek вместо Ollama

**Проблема:** Ollama/llama3.2 работал медленно и давал нестабильное качество.

**Решение:** Перейти на DeepSeek API (`deepseek-chat` для быстрых, `deepseek-reasoner` для сложных задач).

**Ключ:** `~/.aim_env → DEEPSEEK_API_KEY`

**Обратная совместимость:** `llm.py` поддерживает fallback на Ollama если DEEPSEEK_API_KEY не задан.

---

### 2026-03-28: Мульти-тенантность с самого начала

**Решение:** Архитектура с изоляцией тенантов с первого дня.

**Причина:** Планируется масштабирование на несколько клиник (FCLC, WLRAbastumani, DrJaba).

**Схема:** Каждый тенант получает отдельную схему БД `aim_tenant_{id}`. В SQLite — отдельный файл БД на тенант.

### 2026-03-28: RBAC с 15+ ролями

**Роли:** PATIENT, FAMILY_MEMBER, GUARDIAN, DOCTOR, NURSE, LAB_TECHNICIAN, ADMINISTRATOR, INSTITUTION_ADMIN, DEPARTMENT_HEAD, LAB_PARTNER, PHARMACY_PARTNER, INSURANCE_PARTNER, SYSTEM_ADMIN, AUDITOR.

**Матрица прав:** Полностью описана в CONCEPT.md §3, реализована в `core/rbac.py`.

## Известные проблемы и их решения

### Проблема: OCR русского текста в анализах

**Решение:** Tesseract с `rus+kat+kaz+eng`, fallback на RapidOCR.
Для повышения качества: предобработка изображений (контрастность, DPI 300+).

### Проблема: Идентификация пациентов в WhatsApp

**Решение:** Формат имени контакта: `SURNAME P|П|პ FIRSTNAME`. Разделитель P/П/პ автоматически детектируется.

### Проблема: Производительность при большом количестве пациентов

**Решение:** SQLite WAL mode, индексы на patient_id + date. В production — PostgreSQL + connection pool.
