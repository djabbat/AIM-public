## 1. Что такое AIM

AIM — локальная AI-система для ведения пациентов интегративной медицины.

**Работает полностью локально** — никакие данные не покидают компьютер.

**Что делает:**
- Хранит историю болезни каждого пациента
- Делает OCR скринов с анализами (фото с телефона)
- Извлекает данные из PDF лабораторий
- Автоматически строит дифференциальный диагноз (Байесовская модель)
- Генерирует интегративные рекомендации через AI (llama3.2)
- Синхронизируется с WhatsApp Desktop автоматически

**Языки:** русский, грузинский, казахский, английский

## 4. Импорт пациентов из WhatsApp

### Метод А: Автоматическая синхронизация (рекомендуется)

**Шаг 1.** Убедитесь, что контакты пациентов названы правильно (см. раздел 9)

**Шаг 2.** Запустите WhatsApp Desktop (после первого запуска с флагом — он запускается автоматически с отладочным портом)

**Шаг 3.** В AIM нажмите кнопку **`⚡ Sync WhatsApp`**

Или в терминале:
```bash
python3 wa_extractor.py
```

Система автоматически:
- Найдёт все чаты с пациентами (по разделителю П/P/პ)
- Извлечёт переписку
- Запустит полную обработку

---

### Метод Б: Ручной экспорт

**На Android:**
1. Открыть чат → ⋮ (три точки) → **Ещё** → **Экспорт чата**
2. Выбрать **"Прикрепить медиафайлы"**
3. Отправить себе (Telegram / почта)

**На iPhone:**
1. Открыть чат → имя контакта → **Экспорт чата**
2. Выбрать **"Прикрепить медиафайлы"**
3. Отправить себе

**Получится файл:** `WhatsApp Chat with Robakidze П Nino.zip`

**Шаг:** Скопировать `.zip` в `~/AIM/Patients/INBOX/`

**Запустить импорт:**
```bash
python3 patient_intake.py   # обработает всё из INBOX
```
Или в GUI: кнопка **`↑ Импорт файлов`**

## 6. Обработка данных

### OCR скринов (фото анализов с телефона)
- Движок: **Tesseract** (рус + груз + англ)
- Запасной: **RapidOCR** (автоматически)
- Результат: `FILENAME_ocr.txt` рядом с изображением
- Повторная обработка: `--force`

### Извлечение PDF
- Движок: **pdfplumber**
- Результат: `FILENAME_text.txt`

### AI анализ
- Модель: **llama3.2** (локально через Ollama)
- Результат: `_ai_analysis.txt`
- Содержит: жалобы, анамнез, отклонения лабораторий, диагноз, рекомендации

### Принудительный пересчёт всего
```bash
python3 patient_intake.py --all --force
```

## 8. WhatsApp Auto-Sync

### Как это работает
```
WhatsApp Desktop → CDP (порт 9222) → JS-инъекция → извлечение чатов → INBOX → intake pipeline
```

WhatsApp Desktop запускается с флагом `--remote-debugging-port=9222`.
AIM подключается через Chrome DevTools Protocol и читает данные через внутренний JavaScript API WhatsApp Web.

### Первый запуск (уже сделано автоматически)
Launcher обновлён: `/home/oem/whatsapp-desktop-linux/data/io.github.mimbrero.WhatsAppDesktop.desktop`
Флаг `--remote-debugging-port=9222` добавлен.

**Важно:** После обновления launcher нужно **перезапустить WhatsApp**.

### Ограничения Auto-Sync
- WhatsApp должен быть запущен и авторизован
- **Медиафайлы** (фото анализов) через CDP не скачиваются — их нужно пересылать вручную в папку пациента или экспортировать через "Экспорт чата с медиа"
- WhatsApp периодически обновляет внутренний JS API — если sync перестал работать, используйте Метод Б (ручной экспорт)

### Команды
```bash
python3 wa_extractor.py                  # синхронизировать всех пациентов
python3 wa_extractor.py --list-only      # только показать найденных пациентов
python3 wa_extractor.py --no-import      # извлечь без запуска анализа
python3 wa_extractor.py --patch-launcher # обновить launcher (уже сделано)
```

## 9. Формат имён контактов

Чтобы AIM распознал контакт как пациента, имя должно содержать разделитель:

```
ФАМИЛИЯ [П | P | პ] ИМЯ
```

| Разделитель | Язык | Пример |
|-------------|------|--------|
| `П` | Русский | `Robakidze П Nino` |
| `P` | Латинский | `Beridze P Keti` |
| `პ` | Грузинский | `ნისhnianidze პ ანი` |

**Правила:**
- Фамилия и имя — теми символами, которыми записано в контакте
- Папка создаётся с теми же символами: `Robakidze_Nino_...`
- Дата рождения определяется автоматически из анализов
- Если пациент добавил родственника в чат — система ищет ДР в тексте анализов

## 10. Технические детали

### Зависимости
```bash
# Python пакеты (в venv)
pip install ollama pdfplumber pytesseract rapidocr-onnxruntime \
            websocket-client requests Pillow python-docx deep-translator

# Системные пакеты
sudo apt-get install tesseract-ocr tesseract-ocr-rus tesseract-ocr-kat tesseract-ocr-kaz
```

### Модели Ollama
```bash
ollama serve
ollama pull llama3.2     # основная модель (2GB)
ollama pull deepseek-r1:7b  # альтернатива для сложного анализа
```

### Смена модели
В файле `~/AIM/medical_system.py`, строка:
```python
MODEL = "llama3.2"
```
Поменять на `"deepseek-r1:7b"` для более глубокого анализа.

### Саморазвивающаяся база знаний
`~/AIM/medical_knowledge.json` — накапливается автоматически при каждом анализе пациента.
Хранит: частотность диагнозов, эффективные протоколы, паттерны по группам пациентов.

### Резервные копии
```bash
cp -r ~/AIM/Patients ~/AIM/Patients_backup_$(date +%Y%m%d)
```
