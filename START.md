## ПРАВИЛО 4 — ДВА CLAUDE РАБОТАЮТ ПАРАЛЛЕЛЬНО

**Важно:** С проектами работают два экземпляра Claude Code одновременно.
Один может не знать, что сделал другой в прошлой сессии.

**При возникновении сомнений — СПРАШИВАТЬ пользователя:**

Примеры ситуаций, когда нужно спросить:
- Файл существует, но содержание неожиданное (другой Claude мог изменить)
- Задача в TODO помечена как выполненная, но файлы не найдены
- Конфликт между состоянием файла и записями в прогресс-JSON
- Непонятно, на каком этапе находится многошаговая задача
- Найден неожиданный файл/ветка/изменение

**Формула вопроса:**
> *"Я вижу [ситуация]. Возможно, второй Claude уже работал с этим. Что именно нужно сделать?"*

**Никогда не перезаписывать и не удалять без явного подтверждения.**

## ПРАВИЛО 8 — GIT PUSH: ВСЕГДА СПРАШИВАТЬ PRIVATE ИЛИ PUBLIC

**При каждом git push любого проекта:**

1. Спросить пользователя:
   > _"Пушить в **private** или **public** репозиторий?"_

2. Проверить, что оба репозитория существуют:
   - **Private:** закрытый (код, данные, черновики)
   - **Public:** открытый (публикуемая версия, open-source)

3. Если один из репозиториев отсутствует — **создать его** через `gh repo create`:
   ```
   gh repo create OWNER/REPO-private --private
   gh repo create OWNER/REPO-public  --public
   ```

4. Добавить нужный remote если не существует:
   ```
   git remote add private git@github.com:OWNER/REPO-private.git
   git remote add public  git@github.com:OWNER/REPO-public.git
   ```

5. Пушить только в выбранный.

**Текущий статус AIM:**
- `origin` → `git@github.com:djabbat/AIM.git` (статус private/public — уточнить у пользователя)
- Второй репозиторий — создать по запросу
