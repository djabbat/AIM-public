## ПРАВИЛО 2 — ЧИТАТЬ СОБСТВЕННОЕ TODO

**При каждом запуске прочитать:**

```

```

Обратить особое внимание на:
- Незавершённые задачи с пометкой `[ ]`
- Критические баги / безопасность (zip bomb, race conditions)
- Интеграционные модули в статусе 🟡

## ПРАВИЛО 4 — ДВА CLAUDE РАБОТАЮТ ПАРАЛЛЕЛЬНО

**Важно:** С проектами работают два экземпляра Claude Code одновременно.
Один может не знать, что сделал другой в прошлой сессии.

**При возникновении сомнений — СПРАШИВАТЬ пользователя:**

Примеры ситуаций, когда нужно спросить:
- Файл существует, но содержание неожиданное (другой Claude мог изменить)
- Задача в TODO помечена как выполненная, но файлы не найдены
- Конфликт между состоянием файла и записями в прогресс-JSON
- Непонятно, на каком этапе находится многошаговая задача
- Найден неожиданный файл/ветка/изменение

**Формула вопроса:**
> *"Я вижу [ситуация]. Возможно, второй Claude уже работал с этим. Что именно нужно сделать?"*

**Никогда не перезаписывать и не удалять без явного подтверждения.**
